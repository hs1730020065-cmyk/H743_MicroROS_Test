CAN/LIN 线控转向自测数据说明
===============================

数据来源：CAN/LIN Tool V6.0 上位机采集
场景：车辆线控转向（EPS）CAN 总线报文记录

---

一、文件命名规则

文件分为两类：

1. 按运动状态命名（转向方向测试）：
   - 不动.csv   —— 车辆静止
   - 前进.csv   —— 直行前进
   - 后退.csv   —— 直行后退
   - 左转.csv   —— 方向盘左转
   - 右转.csv   —— 方向盘右转
   - 按照规律来.csv —— 按预设规律交替运动

2. 按距离命名（距离控制测试）：
   - 正值表示前进方向位移：
       0.3m.csv、0.5m.csv、0.8m.csv、1m.csv、1.5m.csv、2m.csv
   - 负值表示后退方向位移：
       -0.5m.csv、-1.0m.csv、-2m.csv

---

二、CSV 字段说明

| 列名           | 含义                          |
|----------------|-------------------------------|
| 序号           | 报文序号（从 0 开始递增）        |
| 传输方向       | 数据传输方向（"发送"或"接收"）   |
| 系统时间       | 系统时间戳（时:分:秒.毫秒）       |
| 时间戳(ms)     | 距离第一帧的相对时间偏移（毫秒）  |
| ID号(hex)      | CAN 报文 ID（十六进制）          |
| 帧类型         | CAN 帧类型（标准帧/扩展帧）       |
| 帧格式         | CAN 帧格式（数据帧/远程帧）       |
| 长度           | 数据长度（DLC，单位字节）        |
| 数据(hex)      | 8 字节数据载荷（十六进制，空格分隔）|

四、常见 CAN ID 参考

数据中出现的典型 CAN ID（十六进制）：
  - 0x186 —— 可能与角度/扭矩信号相关
  - 0x18F —— 可能与速度信号相关
  - 0x206 —— 可能与状态信号相关
  - 0x286 —— 可能与目标位置信号相关
  - 0x386 —— 可能与传感器反馈相关
  - 0x7B9 —— 可能与诊断/标定信号相关

注：具体 ID 含义请参考《CAN协议完整功能码详解_v4.docx》和《线控转向EPS说明文档.pdf》。

---

五、使用说明

1. 每条记录为 CAN 总线的一帧报文，数据字段为十六进制原始载荷。
2. 时间戳(ms) 列以每个文件第一帧为 0 点，可用于计算报文间隔和时序分析。
3. 分析时建议配合协议文档对 ID 和数据字段进行解析。
4. 按运动状态的文件适合做工况分析；按距离的文件适合做位移-信号关系分析。
