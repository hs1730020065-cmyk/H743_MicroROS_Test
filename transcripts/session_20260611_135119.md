## 为什么转向角没变？

看 `Ackermann_Calc` 的计算公式（`chassis_control.c:218`）：

```
转向角 δ = atan(L × ω / v)
```

代入你的参数：
- 轴距 L = 0.660 m
- 速度 v = 0.73 m/s (恒定)
- 最大转向角 = 21.2°（机械极限，`chassis_control.c:28`）

**关键计算**：在 v=0.73 时，转向角达到 21.2° 极限只需要多大的 ω？

```
ω_临界 = v × tan(21.2°) / L = 0.73 × 0.388 / 0.660 ≈ 0.43 rad/s
```

而你的 turn 值从 **1.89 → 0.81**，全部远超 0.43 rad/s：

| turn (ω) | 计算转向角 δ | 实际输出 (限幅后) |
|-----------|-------------|-------------------|
| 1.89 | atan(1.71) ≈ 59.7° | **21.2°** (饱和) |
| 1.53 | atan(1.38) ≈ 54.2° | **21.2°** (饱和) |
| 1.00 | atan(0.90) ≈ 42.0° | **21.2°** (饱和) |
| 0.81 | atan(0.73) ≈ 36.2° | **21.2°** (饱和) |

**所有值都被限幅到同一个角度了，所以你看不到转向角变化。**

## 怎么才能看到转向角变化？

有两个办法：

1. **降低角速度命令**：把 turn 降到 **0.43 rad/s 以下**，转向角才会开始脱离饱和区
2. **提高车速**：比如 v=1.5 m/s 时，ω 上限 ≈ 0.88 rad/s，这样 turn=0.81 就能看到不同转角了

简单说：你现在的速度太慢、转弯命令太大，转向角一直被压在机械极限上，所以看起来"没变化"。

## 公式推导

转向角和转弯半径的关系是：

```
tan(δ) = L / R

```

```

代入你的车参数（`chassis_control.c:16,28`）：

```
δ_max = atan(0.660 / 1.7)
      = atan(0.3882)
      ≈ 21.2°
```

- L = 0.660 m → 轴距（前后轮距离）
- R_min = 1.7 m → 最小转弯半径（机械设计决定）

## 直观理解

```
         R_min = 1.7m
    ←──────────────→
         ┌─────┐

             \ 转弯中心

你怎么同步呢，我们这边是串口通信，通过串口相连的时候自动赋值吗

---

## 🤖 Claude

就是走同一条串口线，多传几帧数据而已。

**原理：**

```
STM32  ──串口──>  micro-ROS Agent (PC)
                  │
                  │  Agent 知道 PC 的 ROS 时间
                  │  (就是系统时间，Unix 时间戳)

① STM32 调用 rmw_uros_sync_session(1000)
    → 发一个时间请求帧，记录自己的发送时刻 t1

② Agent 收到请求
    → 立即回一个响应帧，里面带着 agent 侧的当前时间 t_ros

③ STM32 收到响应，记录接收时刻 t2
    → 计算: 实际时间 ≈ t_ros + (t2 - t1)/2
    → 存下这个 offset

④ 之后每次调 rmw_uros_epoch_nanos()
    → 返回 STM32本地时间 + offset = 同步后的 Unix 纳秒时间
```

**不需要额外接线**，就在你现有的串口线上多传两帧。你初始化的时候调一次 `rmw_uros_sync_session`，后面就自动有了。首次同步大概几十毫秒就完成。

**0x206（电机）：** `07 00 00 1D FF 00 00 00`

```
raw = 0.5 × 454.5 = 227
正转补码 = 0x10000 - 227 = 0xFF1D
data[3]=0x1D  data[4]=0xFF  (小端)
```

**0x314（转向）：** `01 01 68 00 00 00 00 00`

```
δ = atan(0.66 × (-1.0) / 0.5) = -52.9°
ROS负号取反 → EPS正值(右转) = 52.9°
限幅 → 30°
EPS = 30 × 12 = 360 = 0x0168
data[1]=0x01  data[2]=0x68  (大端)
```

注意：52.9° 远超 30° 限幅，实际只发 30°。如果不想限幅，把 `CHASSIS_MAX_STEER_ANGLE_DEG` 改大即可。

## 🤖 Claude

一步一步来：

## 1. 阿克曼模型算车轮转角

```
δ_rad = atan(L × ω / v)
      = atan(0.66 × (-1.0) / 0.5)
      = atan(-1.32)
      = -0.9235 rad     ← ROS坐标系：左转为正，右转为负
```

## 2. 方向取反（匹配 EPS 协议）

```
δ_deg = δ_rad × 57.296 × (-1.0)    ← CHASSIS_STEER_DIRECTION_SIGN
      = -52.92 × (-1.0)
      = +52.92°        ← EPS坐标系：正值=右转
```

## 3. 限幅

```
52.92° > 30°  →  限到 30°
```

## 4. EPS 执行器换算（×12 转向比）

```
EPS = 30° × 12.0 = 360
CAN  = 0x0168
帧   = 01 01 68 00 00 00 00 00
```

你 angular.z=-1.0 太大了，52.9° 远超 30° 限幅，所以实际只转到 30°。想让转角脱离限幅，要么降低角速度，要么提高车速。
