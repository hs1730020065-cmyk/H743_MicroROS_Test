## 🤖 Claude

你的代码已经在 STM32 端以 5Hz 频率通过 Micro-ROS 向 ROS2 Agent 发布 `/odom` 话题（`nav_msgs/msg/Odometry`）。查看 odom 数据的方法都在 **PC/ROS2 侧**：

## 1. 命令行查看

```bash
# 查看原始话题数据（实时滚动）
ros2 topic echo /odom

# 只看位置
ros2 topic echo /odom --field pose.pose.position

# 只看速度（twist）
ros2 topic echo /odom --field twist.twist

# 查看话题发布频率
ros2 topic hz /odom
```

## 2. ROS2 基础诊断

```bash
# 确认话题存在
ros2 topic list | grep odom

# 查看话题详细信息（类型、发布者）
ros2 topic info /odom
```

## 3. 图形化查看

```bash
# 用 rqt_plot 画曲线（推荐，直观看到 x/y/theta 变化）
ros2 run rqt_plot rqt_plot /odom/pose/pose/position/x /odom/pose/pose/position/y

# 用 rviz2 可视化（需要设置 fixed frame = "odom"，添加 Odometry display）
rviz2
```

## 4. 你的 odom 发布内容概览

从 `Core/Src/freertos.c:912-957`，你的 odom_timer_callback 每 200ms 发布的数据：

| 字段 | 内容 |
|---|---|
| `header.frame_id` | `"odom"` |
| `child_frame_id` | `"base_link"` |
| `pose.pose.position.(x,y)` | 由实际车速 `vx` 和横摆角速度 `vtheta` 积分得到 |
| `pose.pose.orientation.(z,w)` | 由 yaw 角 `θ` 计算的四元数 |
| `twist.twist.linear.x` | 电机实际车速 `g_drive_motor_actual_speed_mps` |
| `twist.twist.angular.z` | 实际横摆角速度 `g_chassis_actual_angular_z` |

**注意**：目前的 `/odom` 是纯航迹推算（dead reckoning），位置靠 `vx * dt` 积分，没有 IMU 或外部观测做修正，长期会漂移。等 IMU 到货后可用 EKF（如 `robot_localization`）融合后生成更准的 `/odom`。
